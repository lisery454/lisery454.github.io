回退版本

使用reset指令来回退版本，一般有三种方式，为soft，hard，mixed

soft

即回退到某个版本，并且保留当前工作区（untracked）中和暂存区（add后）中的文件修改。

hard

回退到某个版本，并且不保留当前工作区（untracked）中和暂存区（add后）中的文件修改。一般是想要撤销的所有文件操作的时候用

mixed

回退到某个版本，并且保留当前工作区（untracked）中，但是不保留暂存区（add后）中的文件修改。

git ls-files 查看暂存区中内容

如果reset错了也可以撤销，使用reflog指令找到所有操作的历史记录，然后回退到某个历史记录版本就行了。

.gitignore

这个文件可以让我们在使用git时忽略一些工作区中的文件，比如自动生成的文件，日志文件，编译文件等。

.gitignore只能管理没有添加到版本库中的文件，如果要重新管理添加到版本中的文件，使用

git rm --cached file.txt先删除暂存区中的这个文件，再之后commit。
